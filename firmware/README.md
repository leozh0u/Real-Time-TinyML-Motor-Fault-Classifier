# firmware/

STM32CubeIDE project targeting the Nucleo-F401RE. Not yet generated — this is the intended layout.

## Layout

- `Core/Src`, `Core/Inc` — application code: PID motor control (reused from project 1), sensor sampling (ADXL345 SPI, ACS712 ADC, encoder timer), FFT feature extraction call sites, inference call sites, UART telemetry.
- `Drivers/CMSIS-DSP` — FFT / windowing kernels for on-device feature extraction.
- `Drivers/CMSIS-NN` — quantized int8 conv kernels used by the generated inference code.
- `Drivers/BSP` — hand-written drivers for ADXL345 (SPI) and ACS712 (ADC) not covered by HAL.
- `model_c/` — C code generated by STM32Cube AI Studio from the exported ONNX model. Gitignored (regenerate, don't hand-edit) except this README.

## Build order dependency

Firmware inference integration (pipeline step 8) depends on `training/export/` producing a final ONNX model off the quantization tradeoff frontier (step 6). Don't wire up `model_c/` until that's settled.
